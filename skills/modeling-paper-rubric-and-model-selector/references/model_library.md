# 数学建模方法库

用途：当题目路线不明确、需要模型选型、需要给出基线/改进/验证方案时读取。本文件只提供决策框架，最终仍以题面任务、数据条件、时间约束和可解释性为准。

## 选型原则

1. 先把每一问转成可计算任务：输入、输出、指标、约束、可行域。
2. 每问至少给出一个能跑通的经典基线，再给一到两个改进模型。
3. 优先选择能解释、能验证、能在比赛时间内复现的模型。
4. 复杂模型必须说明相对基线的收益：精度、鲁棒性、约束满足、解释力或计算效率。
5. 论文中不要只写模型名称，必须写清目标函数、变量、约束、求解流程、指标与结果解释。

## 任务类型到模型

### 预测与参数估计

适用：趋势预测、未来值估计、影响因素回归、参数反演。

- 基线：线性回归、岭回归、Lasso、移动平均、指数平滑、ARIMA。
- 改进：随机森林、XGBoost/LightGBM、SVR、GAM、状态空间模型、LSTM/Transformer。
- 验证：留出法、K 折、滚动回测、RMSE、MAE、MAPE、残差图、置信区间。
- 风险：样本少时慎用深度学习；非平稳序列需差分、去趋势或分段建模。

### 分类与识别

适用：风险等级、类别判定、是否发生、目标识别。

- 基线：逻辑回归、朴素贝叶斯、KNN、决策树。
- 改进：随机森林、XGBoost/LightGBM、SVM、代价敏感学习、集成学习。
- 验证：准确率、F1、AUC、PR 曲线、混淆矩阵、阈值敏感性。
- 风险：类别不均衡时不要只看准确率，需看 F1、召回率或 PR-AUC。

### 综合评价与排序

适用：多指标评价、择优、排名、综合指数。

- 基线：标准化 + 加权和。
- 改进：熵权法、CRITIC、AHP、组合赋权、TOPSIS、VIKOR、灰色关联分析、PCA。
- 验证：权重敏感性、排名稳定性、与已知事实或专家判断对照。
- 风险：主观权重需要解释来源，客观权重需要解释指标含义是否被扭曲。

### 优化、调度、选址与路径

适用：成本最小、收益最大、资源分配、路径规划、选址、生产调度。

- 基线：线性规划、整数规划、动态规划、最短路、最小生成树、贪心策略。
- 改进：多目标规划、鲁棒优化、随机规划、遗传算法、模拟退火、粒子群、蚁群、禁忌搜索。
- 验证：可行性检查、约束违背率、目标值对比、求解时间、极端场景测试。
- 风险：启发式算法必须给出编码方式、适应度函数、约束处理与收敛/稳定性证据。

### 聚类、分群与画像

适用：无标签分组、用户画像、区域类型划分、模式发现。

- 基线：K-means、层次聚类。
- 改进：GMM、DBSCAN、谱聚类、聚类集成。
- 验证：轮廓系数、Calinski-Harabasz、Davies-Bouldin、稳定性、群体特征解释。
- 风险：聚类结果必须能被业务/题意解释，不能只报算法指标。

### 机理、仿真与系统动力学

适用：机制解释、物理/生态/经济过程、情景推演。

- 基线：差分方程、微分方程、守恒关系、系统动力学。
- 改进：参数校准、贝叶斯估计、蒙特卡洛仿真、数据驱动校正、混合机理模型。
- 验证：历史拟合、参数敏感性、情景一致性、极端条件合理性。
- 风险：所有参数必须有来源、估计方法或敏感性说明。

### 网络、图论与传播

适用：关系网络、交通网络、供应链、影响传播、社区结构。

- 基线：中心性、最短路、最大流/最小割、PageRank、社区发现。
- 改进：复杂网络指标、鲁棒性分析、传播模型、图优化。
- 验证：节点/边删除实验、连通性、效率指标、传播范围对比。
- 风险：网络构建规则必须清楚，否则后续指标没有可信基础。

### 不确定性、风险与鲁棒性

适用：参数不确定、数据噪声、情景预测、风险决策。

- 方法：蒙特卡洛、Bootstrap、置信区间、敏感性分析、鲁棒优化、CVaR。
- 验证：扰动幅度、结果分布、最坏情景、关键参数排序。
- 风险：不要只写“进行了敏感性分析”，必须说明扰动范围和结论变化。

## 组合路线模板

- 预测 + 优化：先预测需求/参数，再把预测值输入优化模型；需补充预测误差对优化结果的敏感性。
- 评价 + 聚类：先构建综合指标，再按指标或原始特征分群；需解释每群的实际含义。
- 机理 + 数据驱动：先用机理模型保证可解释，再用回归/机器学习修正残差；需避免黑箱覆盖机理逻辑。
- 图论 + 优化：先构建网络，再做路径、选址、流量或韧性优化；需明确节点、边、权重定义。

## 论文落地检查

- 每个模型都能对应一个论文小节、一个公式或算法流程、一个结果表/图、一个验证指标。
- 每个结论都回到题目中的某一问，避免“模型很高级但没有回答问题”。
- 每个外部数据或文献都要在数据说明或参考文献中出现。
